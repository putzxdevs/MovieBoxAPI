MovieBoxAPI

Unofficial Python client library for the MovieBox streaming platform. Browse, search, and stream movies & TV shows with full dubbing and subtitle support.

⚡ Quick Start GitHub 📦 PyPI
🏠

Homepage & Catalog

Fetch homepage sections, browse movies, TV series, and animations with pagination and filters.

🔍

Search & Detail

Keyword search with full metadata: cast, genres, seasons, episodes, trailer, dubs, and ratings.

🔊

Multi-Dub Streaming

Each dub has its own subjectId. Switch between 12+ audio languages with different stream URLs.

💬

Subtitles

Auto-fetch subtitles from inline + dedicated endpoints. Supports 13+ languages per title.

🔐

HMAC-MD5 Signing

Full V3 mobile API signing implementation with canonical strings and key rotation.

🔄

Host-Pool Failover

Automatic failover across multiple API hosts for maximum reliability.

📦 Installation

Install from PyPI using pip:

bash 
# Install from PyPI
pip install movieboxapi

# Or with SOCKS5 proxy support (recommended)
pip install "movieboxapi[socks]"

# Or install from source
git clone https://github.com/putzxdevs/MovieBoxAPI-Python.git
cd MovieBoxAPI-Python
pip install -e .
💡
Requires Python ≥ 3.10 and httpx[socks] ≥ 0.27.0. The SOCKS extra is needed if you use a proxy for geo-blocked servers.

⚡ Quick Start

python 
from movieboxapi import MovieBoxClient

# Initialize — region: ID (Indonesia), IN (India), US
client = MovieBoxClient(region="ID")

# Optional: SOCKS5 proxy for geo-blocked servers
# client = MovieBoxClient(region="ID", proxy="socks5://127.0.0.1:40000")

# Search for a movie
results = client.search("Interstellar")
item = results.items[0]
print(f"Found: {item.title} ({item.year})")

# Get full detail
detail = client.get_detail(item.detail_path)
print(f"Rating: ⭐ {detail.rating}")
print(f"Genres: {', '.join(detail.genres)}")

# Get stream
stream = client.get_stream(
    subject_id=detail.subject_id,
    se=1, ep=1,
    resolution=1080
)
print(f"Video: {stream.url}")
print(f"Subtitles: {len(stream.subtitles)} tracks")

🔧 MovieBoxClient

The main entry point. Create an instance and use its methods to interact with MovieBox.

python 
from movieboxapi import MovieBoxClient

client = MovieBoxClient(
    region="ID",       # ID, IN, US
    host="moviebox.ph",  # Frontend hostname
    proxy=None,          # "socks5://127.0.0.1:40000"
    timeout=25.0,       # HTTP timeout (seconds)
)
Parameter Type Default Description
region str "ID" Region code (ID=Indonesia, IN=India, US)
host str "moviebox.ph" Frontend hostname for H5 API headers
proxy str | None None SOCKS5/HTTP proxy URL
timeout float 25.0 HTTP request timeout in seconds

🏠 Home

GET client.get_home() Fetch homepage sections ▶

Returns a list of HomeSection objects (banner, movie, tv, animation, sport_live).

Returns: list[HomeSection]

python 
sections = client.get_home()
for section in sections:
    print(f"[{section.type}] {section.name} — {len(section.items)} items")
    for item in section.items[:3]:
        print(f"  • {item.title} (⭐ {item.rating or 'N/A'})")

🔊 Dubs & Audio Languages

Each dubbing language has its own subject_id. Use the Dub objects from detail.dubs to switch audio languages.

✅
Different subjectId = Different stream URL = Different audio track. This is confirmed working with 12+ dub languages.
python 
detail = client.get_detail("/id/tv/avatar-the-last-airbender-abc123")

# Filter dubs (type=0) and subs (type=1)
dubs = [d for d in detail.dubs if d.type == 0]
subs = [d for d in detail.dubs if d.type == 1]

# List available dubs
for dub in dubs:
    marker = " (original)" if dub.original else ""
    print(f"  {dub.lan_code}: {dub.lan_name} — id={dub.subject_id}")

# Stream original audio
stream = client.get_stream(subject_id=detail.subject_id, se=1, ep=1)

# Stream Indonesian dub (different subjectId!)
indonesian = next(d for d in dubs if d.lan_code == "id")
stream_id = client.get_stream(subject_id=indonesian.subject_id, se=1, ep=1)
print(stream_id.url)  # Different URL → different audio!

▶️ Stream

GET client.get_stream(subject_id, se, ep, resolution, lan) Get playable stream ▶

Returns a StreamResult with video URL, qualities, and subtitles. Uses HMAC-MD5 signed V3 API with host-pool failover.

Returns: StreamResult

ParamTypeDefaultDescription
subject_idstrrequiredMovieBox subject ID
seint1Season number
epint1Episode number
resolutionint1080Preferred resolution (360, 480, 720, 1080)
lanstr | NoneNonePreferred audio language code
python 
stream = client.get_stream(
    subject_id="12345",
    se=1, ep=1,
    resolution=1080,
)
print(f"Video URL : {stream.url}")
print(f"Subtitle  : {stream.subtitle_url}")

# Available qualities
for q in stream.qualities:
    marker = " ← current" if q.current else ""
    print(f"  {q.label}", marker)

# Available subtitles
for s in stream.subtitles:
    print(f"  [{s.lang}] {s.label}")
GET client.get_stream_url(subject_id, se, ep, resolution) Shorthand: just the URL string ▶

Returns just the video URL string.

Returns: str

📋 Models

All models are plain dataclasses — no pydantic dependency. Convert to dict with .to_dict() or dataclasses.asdict().

Model Description
MediaItemCatalog/search card (title, poster, rating, year)
MediaDetailFull metadata (cast, seasons, episodes, dubs, trailer)
StreamResultPlayable stream (URL, qualities, subtitles)
CatalogPagePaginated catalog response
SearchResultPaginated search response
HomeSectionHomepage section (banner, featured, etc.)
QualityVideo quality option (360p–4K)
SubtitleSubtitle track (lang, label, URL)
Season / EpisodeSeason and episode numbers
CastMemberCast/crew member (Director, Actor, Writer)
DubAudio dub / subtitle variant (each has its own subject_id)
TrailerTrailer video info

Dub Fields

FieldTypeDescription
subject_idstrUnique ID for this language variant
lan_namestre.g. "Indonesian dub"
lan_codestre.g. "id", "en", "ar"
detail_pathstrSlug path for the variant's detail page
typeint0 = dub (audio), 1 = sub (subtitle-only)
originalboolTrue if this is the original audio track

ExceptionWhen

APIErrorAPI returns an error response

TokenErrorCannot acquire auth token
python 
from movieboxapi import MovieBoxClient, GeoBlockError, StreamError

client = MovieBoxClient(region="ID", proxy="socks5://127.0.0.1:40000")
try:
    stream = client.get_stream(subject_id="12345", se=1, ep=1)
except GeoBlockError:
    print("Need a proxy for this region")
except StreamError as e:
    print(f"No stream available: {e}")

⚙️ Configuration

Regions

Built-in presets: ID (Indonesia), IN (India), US (United States). Each preset controls: locale, timezone, carrier code, language, and content region.

Proxy

If your server IP is geo-blocked, pass a proxy:

python 
client = MovieBoxClient(
    region="ID",
    proxy="socks5://127.0.0.1:40000",  # WARP, V2Ray, etc.
)

HMAC Secret Rotation

V3 signing keys can be overridden via environment variables:

bash 
export MOVIEBOX_SECRET_KEY_DEFAULT="your_key_here"
export MOVIEBOX_SECRET_KEY_ALT="your_alt_key_here"

🏗️ Architecture

Two API Layers

LayerHostAuthPurpose
H5 (web) h5-api.aoneroom.com Bearer token from x-user header Home, catalog, search, detail metadata
V3 (mobile) api3-8.aoneroom.com + SG variants HMAC-MD5 signed headers + Bearer Streaming URLs, subtitles

Why Redmi Devices?

📱
The V3 mobile API expects requests from the official Android app (com.community.oneroom). The library generates randomized device fingerprints using Redmi phone models. These are NOT mirrors or proxies — they are spoofed User-Agent and X-Client-Info headers that make requests look like they come from a real phone. This is necessary because the API rejects requests that don't match the expected app signature.

Request Signing

Every V3 request is signed with HMAC-MD5:

  1. Build a canonical string: METHOD\nAccept\nContent-Type\nBodyLength\nTimestamp\nBodyHash\nCanonicalURL
  2. HMAC-MD5 the canonical string with the shared secret (base64-decoded)
  3. Encode the MAC as base64
  4. |2|\"" data-id="Header: x-tr-signature: \"|2|\""> Header: x-tr-signature: "|2|"
